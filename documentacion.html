Sistema de Apoyo al Cuidado Postoperatorio Cardiaco

Resumen Ejecutivo

Este sistema está diseñado para asistir el proceso de recuperación de pacientes postoperatorios cardíacos, proporcionando acceso a recursos educativos, facilitando la comunicación efectiva entre pacientes, familiares y profesionales clínicos, y optimizando la gestión y seguimiento de citas médicas. La solución incorpora arquitectura modular, protección de datos personales, y está alineada a buenas prácticas en accesibilidad y usabilidad.

Contexto y Problema

Los pacientes postoperatorios cardíacos presentan un riesgo elevado de complicaciones debido a dificultades en el seguimiento clínico, deficiente comunicación y baja adherencia al plan de cuidados. La falta de accesibilidad a información confiable y la limitada conectividad con el equipo médico obstaculiza la detección temprana y el soporte continuo.

Necesidad principal:
  • Acceso centralizado a información de cuidados tras la cirugía cardíaca
  • Canal digital para resolver dudas y facilitar la gestión con profesionales clínicos
  • Herramienta amigable para agendar y gestionar citas de seguimiento

Análisis del Entorno

  • Entorno clínico: Hospitales, clínicas y centros de rehabilitación con recursos profesionales limitados y sistemas fragmentados
  • Diversidad tecnológica: Pacientes y familiares con acceso a smartphone, pero variable en alfabetización digital
  • Conectividad: Diseño robusto para operación local con sincronización flexible

Stakeholders

Paciente

Usuario principal, accede a recursos educativos y comunicación médica.

Familiar/Cuidador

Apoya el cumplimiento del plan de cuidados y utiliza el canal digital para resolver inquietudes propias y del paciente.

Personal Médico

Resuelve consultas, gestiona citas y monitorea el proceso de recuperación.

Administrador

Gestiona usuarios, contenidos y supervisa el correcto funcionamiento de la plataforma.

Metodología de Desarrollo

Se utiliza Scrum adaptado para equipos reducidos, con ciclos iterativos quincenales. Incluye integración sistemática de feedback de usuarios y trazabilidad de requisitos. La priorización dinámica permite ajustes ágiles ante nuevas necesidades identificadas durante el semestre.

Arquitectura Técnica Propuesta

Frontend

  • Aplicación web responsive orientada a pacientes y familiares
  • Panel administrativo para profesionales médicos
  • Diseño accesible y centrado en experiencia de usuario

Backend

  • API REST para gestión de usuarios y citas
  • Control de autenticación y roles
  • Módulo de comunicación interna

Base de datos

  • Estructura normalizada para usuarios, consultas y comunicaciones
  • Gestión eficiente de registros y auditoría de actividad

Funcionalidades Clave

Portal de información educativa:

Acceso a guías y materiales de apoyo sobre cuidados postoperatorios.

Comunicación directa:

Mensajería segura entre paciente, familia y personal médico.

Gestión de citas médicas:

Solicitud, confirmación y seguimiento de citas desde la plataforma.

Panel clínico:

Visualización del estado de los pacientes y sus solicitudes.
